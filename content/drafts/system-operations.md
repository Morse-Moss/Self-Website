## 一句话

一条打通「素材采集 → 内容库 → AI 改写创作 → 图片润色 → 审核发布 → 定时调度」的 AI 内容生产流水线。让内容运营从「在五个工具之间复制粘贴」变成「一个界面走完全程」。

## 痛点

内容运营的日常是碎的:找选题用一个工具,存素材用 Excel 和文件夹,改写要把内容贴到 AI 平台再贴回来,修图开 Photoshop,发布再手动登录创作者平台上传。每一步都不难,但串起来每天要消耗大量机械操作时间——这正是「重复超过 3 遍」的教科书场景。

## 方案与架构

```
热点与关键词洞察(实时热词、关键词组管理)
  → 内容发现(关键词搜索、多维筛选、详情预览)
  → 内容库(统一沉淀、标签管理、卡片/列表双视图、导出)
  → 草稿工坊(三栏编辑器:草稿队列 + 编辑器 + AI 助手;一键改写正文/润色标题/生成标签)
  → 图片工坊(AI 图片润色、参考图、原位替换)
  → 发布中心(发布校验、立即/定时发布、状态追踪)
  → 定时调度(周期性内容生产管线、任务中心全量审计)
```

技术栈:FastAPI + SQLAlchemy/Alembic 后端,React + Vite + Ant Design 前端。后端自动化测试 159 个全部通过(2026-06 基线)。模型层支持任意 OpenAI 兼容 API,供应商可配置切换。

几个设计决策:

- **内容库属于用户,不属于某个平台账号**:采集的素材统一沉淀成用户自己的内容资产,平台只是发布出口。
- **AI 改写在编辑器内完成**:不用把内容搬出去再搬回来。标题、正文、标签、配图在同一个三栏工作区内闭环。
- **数据洞察反哺创作**:仪表盘提供互动趋势、Top 内容、热门话题、评论分析,让「下一篇写什么」有数据依据。
- **克制的自动化边界**:平台操作默认低频、串行;敏感凭据全部加密存储,定期健康巡检、过期主动通知;真实账号的发布动作需要明确授权,开发验证一律走测试路径。项目规则里白纸黑字写着:不为绕过平台规则实现任何规避机制。**效率来自流程打通,不来自对抗平台。**
