## 推荐对外简介

一套把深度研究从“AI 写报告”升级为“证据驱动的多 Agent 研究生产线”的本地优先系统。它会完成方法规划、并行检索、证据整理、关键论断映射、受限修复和双重审校；证据不足就停止发布，审核通过后再输出 Markdown / Word。

系统全部技术实现由我独立完成，包括 Agent 编排、前后端、数据与任务运行时、模型与工具接入、质量门和测试体系；研究场景、业务需求和产品方向来自真实协作对接，不表述为完全由我一人设计。

## 首层只展示四个亮点

- **多 Agent 分工协作**：规划、研究、来源审查、综合、验证、修复和主编通过任务、事件与工件协作，不依赖不可观察的隐藏对话。
- **多源研究能力**：接入 OpenAI-compatible 模型中转、DuckDuckGo、Static HTTP、Jina Reader、AnySearch 与 Browser CDP，并设置 URL、安全与降级边界。
- **证据不足不出厂**：关键论断必须绑定可用证据；发现缺口后最多进行两轮受限补采与修复，仍不满足就输出诊断结果而不是伪装成功。
- **可观察、可恢复、可审批**：工作台展示队列、DAG、工件、评审和报告；支持节点边界暂停/恢复、采集结果复用、双 Agent 审校与人工发布批准。

## 详情页叙事

```text
研究问题
  -> 方法规划与任务拆分
  -> 纵向 / 横向并行研究
  -> 搜索、抓取、证据笔记与来源台账
  -> 报告草稿与关键论断-来源映射
  -> 质量评审与受限缺口修复
  -> 证据评审员 + 主编双重验收
  -> 人工批准后发布 Markdown / Word
```

最值得展示的真实案例证据：医学研究与 AI 资产管理两个主题已完成真实端到端验收；其中医学主题识别出证据缺口，执行一次受限修复并重新通过评审后才允许发布。这个案例比单纯强调 Agent 数量更能证明系统能力。

工程能力可以收束为三句话：

- 用 FastAPI / Pydantic、React / TypeScript / XYFlow 完成前后端和可视化工作台。
- 用类型化任务、事件、工件、Tool Gateway、质量门和人工批准构建可审计运行时。
- 用节点租约、暂停恢复、采集检查点和失败诊断处理长链路任务的可靠性问题。

## 未来方向

把系统继续演进为受控自进化的 Agent OS：从高质量研究中提炼可复用的方法和 playbook，经人工审核后进入长期知识，再在后续任务中结构化复用；同时逐步补齐语义事实核验、系统级修复编排和更完整的分布式执行。

“自进化”只作为方向，不写成已完成能力。当前不宣称系统能够自动改代码、绕过人工审批写长期记忆，或完成语义层面的事实真伪判断。

## 不能公开夸大的边界

- 已验收的是固定 `hv_analysis` 报告生产链，不代表通用 Agent OS 已完成。
- 当前论断检查证明证据覆盖和词法对齐，不等于语义事实核验。
- Docker 本地运行与 worker 原型不等于分布式多机生产系统。
- Production Memory 已有实验性的人审与材料化链路，但尚未完成产品验收，也不存在绕过授权的自动长期记忆写入。
- 回放只能展示状态、工具事件、工件和评审结果，不能包装成 Agent 的隐藏思维过程。
