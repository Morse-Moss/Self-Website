## 展开详情

### 01 项目简介

本地优先的多 Agent 深度研究与报告生成系统。用户提交研究问题后，系统通过固定 `hv_analysis` 工作流完成领域方法发现、纵向时间线研究、横向对比研究、来源审查、交叉综合、报告起草、论断映射、缺口修复和最终审校。

正式报告经过证据支持检查、质量评估、Evidence Reviewer、Chief Editor 和人工发布审批后开放。Operator Workbench 集中展示运行队列、任务 DAG、治理状态、证据工件和报告，并支持 Markdown 阅读与 DOCX 下载。

### 02 核心能力

- 方法发现：根据研究领域匹配 Domain Profile 与 Domain Playbook，生成研究方法和查询补充。
- 多 Agent 协作：Planner、Researcher、Synthesizer、Evaluator、Finalizer 通过类型化工件衔接任务。
- 多源证据采集：通过搜索、静态抓取、内容提取和 Browser CDP 获取研究资料。
- 论断与证据治理：维护来源台账、证据笔记和 `claim-source-map`，检查关键论断的证据支持。
- 受限缺口修复：根据评估结果执行最多两轮补充检索、证据合并和报告重写。
- 发布治理：通过双 Agent 审校、候选报告晋级和人工发布审批控制正式出口。
- 可观察运行：展示节点状态、事件、工件、暂停恢复、会话检查点和 Worker 诊断。

### 03 系统架构

主流程：

> 研究问题 -> 分类与方法发现 -> 纵向 / 横向并行研究 -> 来源审查与交叉综合 -> 报告草稿 -> 论断映射与质量评估 -> 缺口修复 -> 最终报告 -> 双 Agent 审校 -> 人工发布审批 -> Markdown / DOCX

模块：`工作台层` `编排层` `研究与证据层` `审核发布层` `工件与运行态层`

## 知识库

### 项目定位与价值

深度研究 Agent 系统将研究方法、网络检索、多 Agent 分工、证据治理和报告发布组织成一条可观察工作流，减少长链路研究中的来源丢失、论断失据和完成状态误判。

### 使用流程

用户输入研究问题并选择研究配置；系统完成方法发现和横纵研究，汇总来源与证据，生成并评估报告，修复关键缺口，完成双 Agent 审校和人工发布审批，最后提供正式报告阅读与下载。

### 核心架构

Operator Workbench 负责交互与治理，FastAPI 运行时负责任务编排，研究 Agent 通过 Tool Gateway 获取证据，审核链负责质量与发布决策，运行状态、事件和工件统一保存到本地数据目录。

### 关键技术实现

系统将固定 DAG 与经验证、审批后追加的受限动态节点结合；Agent 通过类型化工件通信；Provider 级联处理网页采集；来源台账和论断映射约束报告；失败任务保留诊断并支持节点边界恢复。

### 个人技术贡献
