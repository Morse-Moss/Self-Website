## 一句话

一套本地优先(local-first)的多 agent 深度研究与报告生成系统:给它一个研究课题,多个 agent 分工完成检索、取证、交叉分析、写作、评审,产出带完整证据链的研究报告——每一条关键论断都必须有可追溯的来源支撑,否则报告不许出厂。

## 痛点

用 AI 做研究最大的问题不是「写不出来」,是「写出来的不敢信」:来源不明、引用编造、结论看着流畅实则悬空。市面上的 deep research 产品把过程藏在黑盒里,你只能选择信或不信。

摩斯的做法是把「不敢信」当成系统要解决的核心问题:**证据不足就不出报告,过程全程留痕可回放。**

## 方案与架构

```
研究请求
  → 意图分类与研究规划(intake_classify → method_discovery)
  → 纵向研究 / 横向研究(可并行;搜索 → 抓取 → 证据笔记 → 来源台账)
  → 报告起草(LLM 撰写,引用纪律强制校验)
  → 论断-来源映射(claim-source mapping:每条关键论断映射到已采集证据)
  → 评审关卡(evaluator:结构、证据质量、关键论断支撑度,含根因标注)
  → 缺口修复(gap repair:证据不足自动补采,最多两轮,有界不失控)
  → 双 Agent 验收(证据评审员 + 主编)
  → 人工发布批准门(人不点头,正式报告接口不放行)
```

工程上的关键设计:

- **证据链是硬闸门,不是装饰**:关键论断在修复后仍无证据支撑,整个 run 直接标记失败并产出诊断报告,而不是硬着头皮出一份看起来完整的报告。「宁可失败,不可编造」写进了状态机。
- **全程事件留痕**:每一步搜索、抓取、LLM 调用、工具执行都记录为结构化事件,支持仪表盘追踪——这也是作品集「试驾间」能做完整回放的基础。
- **可暂停、可恢复**:run 在节点边界可暂停/恢复,后端重启后中断的任务转为暂停态而非失败,已完成的采集工作可复用不重跑。
- **多 agent 分角色、分模型**:编排、规划、研究、综合、评审、主编各角色可配置不同模型档位——高价值判断用强模型,机械执行用便宜模型。
- **正在向 Agent OS 内核演进**:系统正从「报告生成流水线」重构为通用 agent 运行时内核(Goal / Task / 事件账本 / 策略闸门 / 循环执行契约),报告只是其上的第一个应用。

## 一个值得单独讲的细节:能力矩阵

这个项目有一份 `capability-matrix.md`,强制把每项能力标注为四种状态之一:**accepted(已验收)/ implemented-experimental(已实现但实验性)/ planned(已设计未实现)/ not-implemented(明确不做)**。任何文档、任何 AI、任何汇报都不得把实验性能力说成已验收能力。

这是摩斯「诚实工程」哲学的直接体现:AI 时代最便宜的是「看起来能跑的 demo」,最贵的是「敢写清楚自己不能干什么的系统」。
