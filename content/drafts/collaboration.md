## 适合找摩斯的事

**岗位类型(招聘方):**

- AI 工程 / Agent 系统研发:多 agent 编排、agent 运行时、工具链、评估与治理——这是他三套系统的日常
- AI native 产品研发:从 0 到 1 把「AI 承担执行」做成产品闭环(不是在旧产品上贴一个 AI 按钮)
- AI 提效 / 内部工具:把团队的重复工作流 AI 化——他的工作哲学本身就是这个岗位的 JD
- 全栈工程(AI 加持):FastAPI + React 技术栈的完整系统交付,一个人 + AI 顶一条小产线

**项目类型(客户方):**

- 内容生产流水线:从素材到成品的 AI 内容系统(电商内容、运营内容均有实战)
- Agent 系统定制:研究、分析、内容类 agent 工作流的设计与落地
- 交付要求高的场景加分:他做过零环境裸机部署包(客户双击即用)、配套故障字典和现场运行手册

## 和摩斯共事是什么体验

从他给 AI 定的工作规则,可以直接推断他和人共事的风格:

- **结论先行**:先给判断,再给理由,不铺垫背景
- **不谄媚**:方案有问题直接指出,发现更好的做法主动说,不等你问
- **技术决策讲「为什么」和「对用户的影响」**,不堆实现细节
- **模糊需求不空转**:先给最合理的方案,再问要不要调整
- **改完必验证**:test / lint / build 跑过才算改完;交付有检查清单
- **文档驱动**:需求、方案、验收标准先写下来;规范变更先改文档再改实践

## 合作流程建议

1. **先聊数字分身**:网站上的数字摩斯可以做需求初诊——你描述问题,它给初步思路框架,判断匹配度
2. **约真人 30 分钟**:分身筛选后,真人摩斯做深入沟通(需求边界、验收标准、周期报价)
3. **约束先行开工**:项目第一步是写清楚「什么算完成、如何验证」,然后进入迭代交付
