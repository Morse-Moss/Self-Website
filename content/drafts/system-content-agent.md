## 一句话

一套面向电商场景的内容生成 agent 系统:输入商品图和商品信息,产出商品视频、主图套图、详情长图、活动海报、社媒图文——从素材管理到交付审核全流程闭环。已在公司内部上线运行。

## 痛点

电商内容生产是典型的「重复超过 3 遍」场景:同一个商品要出主图、套图、详情页、海报、视频,换一个商品再来一遍。传统做法依赖设计师逐张做图、运营逐条写文案,产能被人头锁死。

## 方案与架构

```
用户(商家/运营)
  → 对话创作 Agent(多轮对话整理创作 brief,判断图片/视频工作流)
  → 工作台(视频生成/图像生成/详情长图/活动海报/社媒图文,多参考图 @ImageN 引用)
  → 统一模型网关(多 Provider 可配置切换:图像/视频/LLM 各自独立选型)
  → 任务运行时(OperationRun:任务追踪、失败重试、状态管理)
  → 商品素材库(Material Pack:素材、提示词版本、生成结果长期沉淀)
  → 作品项目(版本记录 → 审核 → 定稿的交付审核流)
```

技术栈:FastAPI + SQLAlchemy 后端,React + TypeScript + Ant Design 前端。支持公开注册、角色权限、任务级数据隔离。

几个设计决策:

- **模型网关 provider 无关**:图像、视频、LLM 供应商全部是配置项,换模型不动业务代码。模型市场变化快,系统不该跟任何一家供应商结婚。
- **素材库为中心,不是任务为中心**:商家的长期资产是「这个商品的全部素材和生成历史」,不是一次次孤立任务。素材包(Material Pack)是一等公民,生成记录、提示词版本都归档其上。
- **对话创作 Agent 做入口,不做替代**:多轮对话整理需求 → 信息足够且用户确认后才发起生成。Agent 负责降低使用门槛,不剥夺用户控制权。
- **零环境交付**:构建了裸机部署包——客户解压后双击一个 .cmd,自动初始化数据库、启动服务、打开浏览器,不需要安装 Python / Node / 数据库。配套《发布检查清单》《故障字典》和可交给现场 Agent 执行的《部署运行手册》。
