# 首页能力矩阵重设计

> 日期：2026-07-20
>
> 状态：设计已确认，等待书面规格复核
>
> 范围：首页“能力矩阵”内容模型、卡片结构与视觉呈现

## 1. 问题

当前首页从两个精选项目的 `project.capabilities` 中直接拼出能力矩阵，因此展示的是“横纵研究”“证据台账”“BGE + pgvector”“停止与恢复”等具体项目功能。它回答了系统做过什么，却没有回答摩斯具备哪些能够跨项目复用的开发能力。

本轮需要把能力矩阵改成个人能力领域卡片。项目功能继续留在作品卡片和项目详情中，不再充当首页能力矩阵的数据源。

## 2. 目标

- 招聘方能在短时间内判断摩斯的主要技术覆盖与独立交付范围。
- 能力名称使用直接、务实的行业语言，不堆砌“核心、治理、原生、架构”等包装词。
- 每项能力描述“能处理哪类工作”，而不是复述某个项目的功能清单。
- 视觉与现有深空蓝、青色信号和 Warp Tunnel 背景一致，不复制参考图的绿色矩阵雨。
- 保持首页现有作品证据、开发事实和对话入口不变。

## 3. 最终内容

能力矩阵固定为以下五项，顺序不可由精选项目或项目数量改变：

1. **Agent 应用开发**

   把对话、工具调用、任务状态和人工确认串成可运行的 Agent 流程。
2. **全栈开发与部署**

   独立完成前端、后端、数据库、异步任务、权限和服务器部署。
3. **RAG 与知识库**

   完成知识整理、向量检索、来源展示、内容更新和检索效果验证。
4. **多模型与多模态接入**

   接入文本、图片和视频模型，处理不同模型的参数、素材与任务状态。
5. **AI 编程协作**

   结合 Codex、Claude Code、WorkBuddy 完成需求拆解、代码实现、测试与审查，加快复杂项目交付。

本轮不展示项目标签墙、熟练度进度条、评分、雷达图或自我验收表态。可靠性、幂等、恢复、权限和审批等事实继续作为项目工程实现展示，不单独包装成首页能力。

## 4. 数据模型

在 `content/site-content.json` 的 `profile` 下新增独立的 `capabilityMatrix`：

```ts
type ProfileCapability = {
  id: string;
  title: string;
  description: string;
};
```

`lib/site-content.ts` 为该字段增加显式类型。`profile.capabilities` 继续服务现有首屏短标签，不与新矩阵合并。

`MorseHomeSections` 必须从 `content.profile.capabilityMatrix` 渲染卡片，删除从 `featuredProjects.flatMap(project.capabilities)` 推导矩阵的逻辑。项目的 `capabilities` 字段和作品页展示保持不变。

## 5. 页面结构

能力矩阵仍位于“公开代表作品”和“开发事实”之间，沿用当前全宽 band 与内容容器。

标题区：

- 小标题：`CAPABILITY PROFILE`。
- 主标题：`能力矩阵`。
- 辅助文案：`从多个真实项目中沉淀的可复用开发能力。`
- 标题附近使用一段短青色信号线作为结构标记，不增加图标、插画或发光装饰文字。

卡片结构：

```text
01
Agent 应用开发
把对话、工具调用、任务状态和人工确认串成可运行的 Agent 流程。
```

卡片只包含编号、标题和一句说明。卡片本轮保持静态，不增加项目标签、工具标签、按钮或整卡链接。

## 6. 视觉设计

- 桌面端使用两列网格；前四张各占一列，第五张横跨两列。
- 第五张桌面端使用横向内部结构：编号和标题在左，说明在右；字号与其他卡片一致，不把它做成重点宣传卡。
- 卡片使用深空蓝半透明面板、`6px` 圆角、细边框和轻微内侧顶部高光。
- Warp Tunnel 可以透过面板形成低对比空间层次，但正文区域必须保持稳定可读。
- 青色只用于编号、短信号线和局部边线；标题使用高对比文字色，正文使用现有 muted 层级。
- 不使用绿色矩阵雨、大面积外发光、彩色渐变、霓虹描边、浮动卡片或夸张缩放。
- 悬停和键盘焦点只增强边线与内侧反光，不移动卡片、不改变尺寸。
- 卡片不得嵌套其他卡片。

## 7. 响应式与可访问性

- 在桌面容器内维持两列稳定布局，卡片内容变化不得引发布局跳动。
- `760px` 以下切换为单列，第五张恢复普通纵向结构，不再跨列。
- 标题、正文和最长词组在 `390px` 下不溢出，不使用视口宽度缩放字体。
- 矩阵继续使用语义化列表；每张卡片使用列表项与标题层级，不把纯展示卡伪装成链接或按钮。
- 新设计不增加持续动画；`prefers-reduced-motion` 下行为与现有页面一致。

## 8. 验收标准

- 内容合同精确验证五个标题、五条描述和固定顺序。
- 首页能力矩阵不再读取或 `flatMap` 项目 `capabilities`。
- “横纵研究、证据台账、论断映射、缺口修复、发布审批、三类对话工作流、BGE + pgvector、停止与恢复”等项目功能不再出现在首页能力矩阵中。
- 项目卡片和项目详情中的原有能力信息保持不变。
- 1440x900 下为两列布局，第五张横跨两列；390x844 下为单列布局。
- 双宽下无横向溢出、文字遮挡、卡片嵌套或不连贯间距。
- 浏览器控制台和页面错误为 0。
- `npm test` 与 `npm run build` 通过。

## 9. 非目标

- 不调整公开代表作品、项目详情、开发事实、FAQ、聊天入口或简历模式。
- 不新增依赖、外部字体、图标库或运行时资源。
- 不在本轮重新设计首页其他 band。
- 不把暂存的 favicon 候选纳入本轮能力矩阵实现或部署。
