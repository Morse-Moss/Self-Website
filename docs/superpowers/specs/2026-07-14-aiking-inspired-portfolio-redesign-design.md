# Morse Portfolio Redesign

> 日期：2026-07-14
>
> 状态：设计已由 Morse 分段确认，等待书面规格复核
>
> 唯一外部参考：[aiking.dev](https://www.aiking.dev/)
>
> 选定方向：B，保留参考站的交互骨架，转译为 Morse 自有视觉与内容体系

## 1. 产品判断

本项目是面向招聘方、潜在合作方和 Agent 开发同行的 AI 原生个人作品集。网站需要在 30 秒内说明 Morse 是谁、负责开发过哪些系统、技术能力如何被真实项目证明，同时用数字摩斯文字对话展示知识库与智能客服能力。

本轮不复制 aiking.dev 的绿色矩阵皮肤、课程/博客结构、人物形象或内容。只借鉴其固定导航、持续动态背景、滚动渐显、项目卡反馈和明确的首页/作品集分工，并将这些结构转译为深蓝摩斯信号流、真实项目证据和可用的数字摩斯对话。

## 2. 目标与非目标

### 目标

- 将作品集独立为 `/works` 页面，同时让首页保留两个代表项目的快速入口。
- 把首页主身份改为 `Morse`，副标题为 `Agent 系统开发者 × AI Native 实践者`。
- 用低亮度、慢速的深蓝摩斯信号流建立全站连续视觉背景。
- 用两列项目卡与当前页展开详情承载四个复杂项目，不再依赖独立项目详情页。
- 详细展示每个项目的职责、技术栈、架构、工作流、关键决策、证据和公开边界。
- 将数字摩斯作为首屏真实文字对话入口，为后续数字人形象保留可替换的视觉位置。
- 展示 Codex 与 Claude Code 的可追溯开发数据，包括会话、项目、活跃天数和 Token 用量。
- 删除企业内部项目的公开入口、生产截图和可识别部署信息。

### 非目标

- 本轮不开发数字人视频、语音合成、口型、字幕或音频控制。
- 本轮不增加职业经历、课程、博客、静态 FAQ 或虚构联系方式。
- 不生成或使用虚构人物、虚构系统截图、无法追溯的指标或未经确认的项目成果。
- 不照搬 aiking.dev 的颜色、头像、矩阵字符、文案、卡片内容或商业模块。
- 不更改聊天 Provider、RAG 数据库架构或部署方案，除非实现计划另行列明必要兼容改动。

## 3. 目标用户与核心流程

### 目标用户

- 招聘方：快速判断 Morse 的方向、技术覆盖与真实项目责任。
- 潜在合作方：了解可解决的问题，并通过数字摩斯发起需求沟通。
- Agent 开发同行：深入查看架构、工作流、技术决策与公开代码。

### 核心流程

1. 访客进入首页，立即看到 `Morse`、职业定位、个人介绍和数字摩斯文字对话入口。
2. 访客可以浏览两个代表项目、能力矩阵和开发事实，或直接进入 `/works`。
3. 访客在作品集页先看到四个项目全貌，再点击任一项目在当前页面展开详情。
4. 允许公开的项目提供 GitHub；企业内部项目只展示脱敏后的职责、方法和技术事实。
5. 访客可随时通过数字摩斯的引导问题进入文字对话；公开作品不需要邀请码，实时对话仍遵循短期邀请码和既有访问控制。

## 4. 信息架构

### 首页 `/`

1. **固定导航**：不显示品牌名，只保留 `首页 / 作品集`；右侧保留对话与简历模式图标控制。
2. **身份首屏**：主标题 `Morse`；副标题 `Agent 系统开发者 × AI Native 实践者`；下方用一段中文说明 Morse 负责内容创作、自动运营、深度研究和数字分身项目的整体开发。
3. **数字摩斯入口**：当前显示文字对话状态、访客分流、输入和回复；未来数字人素材替换视觉层，不改变对话入口的位置与职责。
4. **代表作品**：仅展示 `深度研究 Agent 系统` 与 `数字摩斯`，并定位到作品集中的对应项目。
5. **能力矩阵**：展示 Agent 系统、RAG 与知识库、多 Agent 工作流、Python/FastAPI、TypeScript/Next.js、PostgreSQL/pgvector 和 AI 辅助开发交付；每项必须关联项目证据，不使用自评百分比。
6. **开发事实**：展示可追溯的对话、项目、活跃天数和 Token 数据，不再使用“杠杆账本”标题。
7. **对话收束**：用招聘、项目、技术栈和合作类引导问题替代静态 FAQ。
8. **页脚**：只保留必要声明与真实 GitHub 入口。

首页不保留完整四项目展厅、职业经历或静态 FAQ。

### 作品集 `/works`

- 导航下方直接进入项目内容，不使用占满首屏的大标题。
- 四个项目默认全部收起；桌面两列，移动端单列。
- 点击项目卡或“展开详情”后，该项目跨两列展开为完整宽度。
- 同一时间只展开一个项目；打开新项目时关闭旧项目。
- 当前展开项目与 URL Hash 同步，例如 `/works#deep-research`。
- 直接访问有效 Hash 时自动定位并展开项目。
- 外部链接按钮独立工作，不触发卡片展开。
- 旧 `/works/[slug]` 地址重定向到 `/works#slug`，不继续承载独立详情页面。

## 5. 作品内容结构

### 收起卡片

- 项目名称、类型和公开状态。
- Morse 在项目中的真实职责。
- 80 至 120 字项目介绍。
- 主要技术栈与 2 至 3 个能力关键词。
- 展开详情按钮。
- 仅在允许公开时显示 GitHub 或其他外部入口。

### 展开详情

1. **项目背景**：解决的问题与开发原因。
2. **我的职责**：明确整个项目开发责任；只有证据确认时才增加整体架构责任。
3. **技术栈**：按前端、后端、数据、AI/Agent 和部署分类，说明技术实际用在什么位置以及关键选型理由。
4. **系统工作流**：用基于真实代码与配置整理的架构图或流程图说明核心路径。
5. **关键决策与证据**：展示实现选择、测试、运行记录、公开仓库或可验证页面。
6. **当前边界**：明确已完成、开发中、实验性和不可公开内容。

技术栈必须从各项目的依赖清单、配置、源码、公开仓库或 Morse 明确确认中核实，不允许跨项目推断。

## 6. 项目公开边界与素材

### 深度研究 Agent 系统

- 可使用公开 GitHub、真实系统截图和经核实的架构信息。
- 不把规划中的能力写成已实现能力。

### 数字摩斯

- 可使用当前网站、文字智能客服、RAG 验证和公开 GitHub 作为证据。
- 数字人形象未完成前不使用任何假头像。

### 内容创作 Agent 系统

- 作为企业内部脱敏案例展示。
- 不展示内网页面、公司信息、客户信息、业务数据、内部地址或源码。
- 没有获批截图时，只使用从本地只读代码核实后绘制并明确标注的“架构示意图”。

### 自动运营 Agent 系统

- 作为企业内部脱敏案例展示。
- 从公开内容中删除现有外部访问域名、生产登录截图、部署平台信息、生产版本标识、访问系统按钮及其他可识别部署信息。
- 不展示公司、账号、运营数据、发布实录或内部实现细节。
- 没有获批截图时，只使用从本地只读代码核实后绘制并明确标注的“架构示意图”。

### 资产原则

- 真实、合法、已获准的素材优先。
- 本轮不使用 image generation 生成项目证据或数字人形象。
- 架构示意必须与真实代码一致，并明确标注为示意图，不伪装成系统截图。
- 素材缺失时展示诚实的无图状态，不使用装饰图冒充项目画面。

## 7. 视觉系统

### 核心视觉

- 全站使用深空蓝背景和低亮度、慢速的摩斯信号流。
- 信号流由点、划、字母和少量技术符号组成，不复制绿色矩阵雨。
- `Morse` 使用青绿色强调色；职业介绍使用高对比白色；正文保持现有深蓝体系的可读层级。
- 数字摩斯对话与未来数字人融入首屏场景，不放入突兀的大型装饰卡片。
- 页面保持首屏克制、内容逐层展开；不堆叠卡片，不使用虚假仪表盘和无意义装饰。

### 导航

- 固定在顶部，初始状态轻量；滚动后增加半透明背景和轻微模糊。
- 当前页面文字高亮，并用短下划线平滑移动表示状态。
- 首页与作品集切换时保留背景层，只替换正文，减少闪白。

### 动效

- 首屏内容约 700ms 分层进入。
- 内容块进入视口时轻微上移并渐显。
- 项目卡悬停约 300ms，只改变边框、辉光和局部位置。
- 项目展开约 450ms，包含高度、透明度和轻位移过渡。
- 动画层不得拦截点击，核心导航和链接使用可靠的原生语义。
- `prefers-reduced-motion` 下停止持续背景、平滑滚动和展开过渡，但保留全部内容与功能。
- 移动端降低 Canvas 密度和更新频率。

## 8. 数字摩斯文字对话

本轮只实现和验收文字对话，不开发视频或语音能力。

### 状态

1. 待机：显示数字摩斯状态和开场问题。
2. 分流：招聘方、合作沟通、同行交流。
3. 验证：开始实时对话时使用短期邀请码；有效会话跳过验证。
4. 对话：支持流式回复、停止、重试、来源入口和作品定位。
5. 异常：邀请码失效、Provider 不可用、超限或知识不足时给出明确原因和下一步。

### 响应式

- 桌面端对话入口融入首屏右侧场景。
- 移动端点击后进入全屏对话。
- 回答涉及项目时提供作品集定位入口。
- 未来数字人只替换视觉呈现层；视频、语音和口型必须在后续独立阶段重新设计与验收。

## 9. 开发事实与统计口径

### 展示指标

- Codex 与 Claude Code 有效会话总数，并提供来源拆分。
- 两个工具覆盖的去重项目数。
- 近 90 天有开发会话的去重自然日。
- Codex 与 Claude Code 的历史累计 Token。
- Codex 与 Claude Code 最近 30 天 Token。
- 可获得时分别展示输入、输出与缓存 Token。
- 统计覆盖起止日期、生成时间和明确的“截至 YYYY-MM-DD”。

### 聚合规则

- Claude Code 会话按顶层会话记录去重；Codex 会话按稳定会话标识去重，活动与归档记录不得重复计算。
- 项目覆盖需要对两个工具的项目标识归一化后去重；输出只保存数量，不保存本地项目路径。
- 活跃天数使用两个工具会话时间戳的日期并集。
- Token 只汇总日志中可验证的 usage 字段，不根据字数、文件大小或模型价格估算。
- 历史记录缺少 usage 时不得补零或推算；页面必须显示实际覆盖日期与缺口说明。
- 聚合脚本只选择会话标识、项目标识、时间戳和 usage 字段，不输出提示词、回复正文、文件内容、密钥或本地路径。
- 构建前重新生成统计文件；页面数据缺失时省略指标，不显示 `0` 冒充真实结果。

## 10. 技术方案

- 沿用 Next.js App Router、TypeScript、CSS Modules 和 `app/styles/tokens.css`。
- 摩斯信号流使用原生 Canvas 2D，不新增 Three.js、GSAP 或动画依赖。
- 滚动进入使用 `IntersectionObserver` 与 CSS transition。
- 作品展开使用客户端状态组件，并同步 URL Hash。
- 作品内容继续由结构化 JSON 驱动，增加技术栈、模块、工作流、证据和公开级别字段。
- 统计脚本扩展为隐私受限的聚合器，并为会话去重、项目去重、时间窗口、Token 缺失和跨工具合并增加测试。
- 内容安全测试必须阻止企业内部域名、生产截图路径、部署平台信息、公司标识、本地绝对路径、密钥和未终审草稿进入公开内容。
- 不安装阶段契约以外的依赖，不写入外部只读项目。

## 11. 错误与边界状态

- Canvas 不可用时回退到静态深蓝纹理，不影响正文或导航。
- 无效项目 Hash 回到四项目收起状态，不显示错误页。
- 项目内容缺少获批图片时显示明确无图状态或已核实架构示意，不生成假图。
- 统计源缺失或部分历史无 usage 时只展示可验证范围和缺口说明。
- 聊天不可用时首页、作品集和公开证据仍可正常使用。
- 长项目名、长技术栈、长 URL、错误消息和移动端全屏对话不得产生横向溢出。

## 12. 验收标准

### 页面与视觉

- 1440px 和 390px 下首页与作品集无溢出、遮挡或不可读文字。
- 首屏在 30 秒内说明 Morse 的身份、方向、代表作品和下一步行动。
- 摩斯信号流不干扰阅读；减少动态效果模式下完全停止。
- 导航固定、当前页面状态明确，页面切换无明显闪白。
- 页面不包含虚构人物、虚构截图、无来源数据或未经确认的成果。

### 作品集

- 四项目默认收起；桌面两列、移动端单列。
- 展开项目跨列显示；打开另一个项目时关闭前一个。
- 有效 Hash 可定位并展开项目，旧详情 URL 可重定向到对应 Hash。
- 外部链接不误触发展开。
- 自动运营 Agent 不出现公开域名、部署平台信息、生产截图或访问按钮。
- 技术栈和项目说明均可追溯。

### 数据与对话

- Codex 与 Claude Code 的历史累计、最近 30 天 Token、总会话、项目覆盖和近 90 天活跃天数可以通过测试夹具复算。
- 页面标明统计截止日期和覆盖范围；缺失数据不冒充零值。
- 文字对话的邀请码、流式回复、停止、重试、来源与错误状态可用。
- 本轮不验收数字人视频和语音。

### 工程质量

- `npm test` 和 `npm run build` 通过。
- 1440 与 390 浏览器实测通过，控制台无意外错误。
- 上线前 Lighthouse 性能达到 90 分以上。
- 无新增未授权依赖，无外部运行时资源。
- `AGENTS.md`、研究草稿、临时截图、输出目录和本地诊断脚本不得进入本轮提交。

## 13. 实施顺序边界

本规格只定义已确认设计，不授权立即开发。Morse 书面复核本规格后，下一步必须先同步 `docs/portfolio-blueprint.md` 的冲突决策，再编写分阶段实施计划。实施计划应将内容脱敏与统计管线放在视觉开发之前，避免受限内容继续进入新界面。
